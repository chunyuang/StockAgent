### P0-6: 🆕 动态滑点实现存在严重的单位计算错误

**文件：** `AgentServer/nodes/backtest_engine/backtester.py` 第340-358行  
**问题：**

1. **volume 单位错误**：`volume` 在数据库中单位是**手**（参见 `akshare_daily_manager.py` 第125行注释 `单位: 手`），但代码第356行 `volume_amount = volume * 10000` 的注释写 `日成交额，单位 万元 → 元`。实际上 `volume * 10000` 是把 手×10000，既不是万元也不是元，结果完全无意义。

2. **circ_mv 是估算值**：`scripts/compute_factors.py` 第132-134行 `circ_mv = amount * 10`，这是基于假设换手率≈10%的粗略估算，不是真实流通市值。动态滑点公式 `slippage / sqrt(circ_mv / 10000)` 高度依赖 circ_mv 精度。

3. **circ_mv fallback 危险**：当 `circ_mv` 字段不存在时默认 `50000`（5亿市值），这意味着大盘股（如平安银行流通市值2000亿）在数据缺失时会被当做小盘股处理，滑点异常放大。

4. **import math 在循环内**：第342行 `import math` 在每个交易日的循环内重复执行，虽然 Python 有 import 缓存不会重新加载，但这违反最佳实践且降低可读性。

**影响：** 
- 卖出冲击成本计算结果完全错误（volume单位混乱）
- 动态滑点基于不准确的估算市值，回测结果可信度存疑
- 数据缺失时回测结果可能严重偏差

**建议：**
1. 使用 `amount` 字段（成交额，单位千元）替代 `volume` 计算冲击成本
2. 获取真实流通市值数据或明确标注为估算
3. 将 `import math` 移到文件顶部
4. circ_mv 缺失时应跳过动态滑点，使用基础滑点
