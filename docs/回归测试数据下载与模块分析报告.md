## 二、数据下载过程分析

### 2.1 原始问题

- 原有脚本 `download_daily_akshare.py` 使用方式错误：`ak.stock_zh_a_daily(symbol=trade_date)`
- AKShare 接口已变更，不支持按日期批量获取，报错 `KeyError: 'date'`
- Tushare 共享节点 `stock_basic` 返回空，无法获取股票列表

### 2.2 数据源选择

| 数据源 | 选择结果 | 原因 |
|--------|----------|------|
| Tushare 共享节点 | ❌ 放弃 | API 返回异常，股票列表为空 |
| AKShare | ✅ 选用 | 开源免费，接口稳定，可获取全市场数据 |

### 2.3 下载方案变更

**原方案:** 按日期下载 → 每个交易日获取全部股票数据  
**新方案:** 按个股下载 → 每个股票获取指定区间数据  

新脚本: `download_daily_akshare_v2.py`

**关键改进:**
1. 通过 `ak.stock_info_a_code_name()` 获取全部 A 股代码 → 共 **5493 只**
2. 代码格式转换：`sh600000` / `sz000001` 符合 AKShare 要求
3. 标准化列名对齐数据库 schema
4. 使用 `bulk_write` + `upsert` 批量写入，自动去重
5. 添加进度条显示实时进度

### 2.4 实际执行过程

- **开始时间:** 2026-03-26 06:44 GMT+8
- **完成时间:** 2026-03-26 09:31 GMT+8
- **总耗时:** ≈ **2小时47分钟**
- **处理速度:** ≈ **1.6-1.9 只/分钟**
- **错误情况:** 少量新股/停牌股票返回空数据，不影响整体完整性

### 4.2 六大策略因子配置

回测脚本已预定义六大游资策略：

| 策略 | 每日选股数 | 因子配置 |
|------|-----------:|----------|
| **半路追涨** | 5 | `limit_up_yesterday` (昨日涨停) + `open_below_limit` (开盘低于涨停) |
| **涨停开板** | 5 | `limit_up_yesterday` + `open_below_limit` |
| **跌停翘板** | 3 | `limit_down_yesterday` (昨日跌停) + `open_above_limit` (开盘高于跌停) |
