## 1. 架构设计

### 1.1 设计原则

1. **单例模式**：所有管理器 (`*Manager`) 都是全局单例，在模块级别创建实例
2. **依赖注入**：通过初始化顺序保证依赖正确
3. **异常隔离**：一个数据源失败自动 fallback 到下一个，不影响整体流程
4. **数据标准化**：不同数据源输出统一格式，上层策略不感知数据源差异
5. **异步优先**：所有 IO 操作使用 asyncio，阻塞操作放到 `run_in_executor`

### 1.2 初始化流程

```python
# 启动顺序 (在 initialize_all_managers 中定义)
1. Redis     → 基础设施
2. MongoDB   → 基础设施
3. Tushare   → 数据源
4. Baostock  → 免费日线数据源 (新增)
5. LLM       → AI 服务
6. Milvus    → 向量数据库
7. Prompt    → 提示词管理
8. Notification → 通知服务
9. Feishu    → 多维表格
```

### 2.1 BaseManager (基类)

**文件**：`core/managers/base.py`

**接口定义**：

```python
class BaseManager(ABC):
    @property
    def is_initialized(self) -> bool
        # 返回是否已初始化
    
    @abstractmethod
    async def initialize(self) -> None
        # 初始化，完成后必须设置 self._initialized = True
    
    @abstractmethod
    async def shutdown(self) -> None
        # 关闭连接，释放资源
    
    @abstractmethod
    async def health_check(self) -> bool
        # 健康检查，返回 True 表示健康
    
    def get_status(self) -> dict
        # 返回状态信息
```

## 3. 策略设计

### 3.1 BaseStrategy (策略基类)

**文件**：`nodes/listener/strategies/base.py`

**核心概念**：

- **Subscription**：策略订阅，一个订阅对应一个策略实例，可以配置参数
- **Snapshot**：当日行情快照，包含涨跌停列表、实时行情、历史K线

**核心接口**：

```python
class BaseStrategy:
    @property
    def name(self) -> str
        # 返回策略名称 (唯一标识)
    
    def evaluate(
        self,
        subscription: StrategySubscription,
        snapshot: DailySnapshot,
        today_data: TodayData,
    ) -> List[StrategyAlert]:
        # 评估策略，返回触发的预警列表
        # subscription: 订阅信息，包含参数
        # snapshot: 当日快照
        # today_data: 今日涨跌停等数据
        # -> 返回空列表表示没有触发
```

### 3.2 数据结构

**StrategySubscription**：

```python
class StrategySubscription:
    id: str              # 订阅ID
    strategy_name: str   # 策略名称
    watch_list: List[str] # 观察列表 (空表示全市场)
    is_all_market: bool  # 是否全市场选股
    parameters: dict     # 策略参数
    enabled: bool        # 是否启用
```

**DailySnapshot**：

```python
class DailySnapshot:
    trade_date: int                          # 交易日期 YYYYMMDD
    limit_stocks: Dict[str, LimitStockInfo] # 涨跌停股票 {ts_code: info}
    quotes: Dict[str, RealTimeQuote]        # 实时行情
    kline_cache: Dict[str, pd.DataFrame]    # 历史K线缓存
```

**LimitStockInfo**：

```python
class LimitStockInfo:
    ts_code: str
    limit_type: int       # 1=涨停, -1=跌停
    limit_price: float    # 涨跌停价格
    pre_close: float      # 昨收
```

**StrategyAlert**：

```python
class StrategyAlert:
    subscription_id: str
    strategy_name: str
    strategy_subtype: str
    ts_code: str
    trigger_price: float
    current_price: float
    trigger_reason: str
    position_ratio: float  # 建议仓位比例 (0-1)
    created_at: datetime
```

### 3.3 limit_open.py (涨停开板/跌停翘板)

**策略文件**：`nodes/listener/strategies/limit_open.py`

**参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| `limit_type` | str | `limit_up`=涨停开板 / `limit_down`=跌停翘板 |

**买入条件**（涨停开板）：

1. 股票昨日涨停
2. 今日开盘价 < 涨停价
3. 属于涨跌停列表（确认昨日涨停）

**入口参数获取**：
- `previous_snapshot` 获取昨日快照
- 在昨日涨跌停列表中找到股票，确认是昨日涨停
- 今日获取开盘价，判断是否开板

---

### 3.4 ma5_buy.py (5日线低吸/龙头低吸)

**策略文件**：`nodes/listener/strategies/ma5_buy.py`

**参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| `ma_type` | str | `ma5`=普通5日线低吸 / `leading`=龙头低吸 |
| `max_distance_pct` | float | 最大距离百分比，(现价 - MA5)/MA5 × 100% |

**买入条件**：

1. 计算5日均线（最近5个交易日收盘价均值）
2. `(current_price - ma5) / ma5` 必须在 `[-max_distance_pct, 0]` 区间
   - 即：价格在5日均线下方，但偏离不超过阈值
   - 龙头低吸允许更大的偏离幅度

**说明**：
- 用于龙头股回踩均线时低吸介入
- 需要至少5天历史日线数据计算均线

---

### 3.5 price_change.py (涨跌幅阈值/半路追涨)

**参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| `threshold_pct` | float | 涨跌幅阈值，绝对值 |
| `direction` | str | `down`=跌幅达到阈值 (低吸) / `up`=涨幅达到阈值 (追涨) |

**买入条件**：

1. 今日开盘涨跌幅 (相对于昨日收盘) 达到阈值
2. `direction=down` 且 `pct < -threshold_pct` → 满足
3. `direction=up` 且 `pct > threshold_pct` → 满足

**常用配置**：
- `threshold_pct=6.0`, `direction=down` → 跌6%低吸

## 4. 涨跌停 fallback 流程设计

### 4.1 完整 fallback 流程

```
获取涨跌停 (get_stk_limit in TushareManager)
│
├─→ 1. 尝试 TuShare API stk_limit
│   ├─→ 成功且非空 → 使用 TuShare 结果 ✓
│   └─→ 失败/空/无权限 → 进入下一步
│
├─→ 2. 尝试 AKShare 获取今日涨跌停
│   │   stock_zt_pool_em (涨停) + stock_zt_pool_dtgc_em (跌停)
│   ├─→ 合并结果非空 → 使用 AKShare 结果 ✓
│   └─→ 结果为空 → 进入下一步 (通常因为超过30天限制)
│
├─→ 3. 尝试 AKShare 获取昨日涨停
│   │   stock_zt_pool_previous_em
│   ├─→ 成功非空 → 使用昨日涨停 ✓
│   │   满足 "昨日涨停今日开盘选股" 需求
│   └─→ 失败/空 → 进入下一步
│
└─→ 4. 尝试新浪爬虫兜底
     ├─→ 成功 → 使用抓取结果 ✓
     └─→ 失败 → 返回空，本日无涨跌停数据
```

### 4.2 关键设计点

1. **异常隔离**：跌停接口获取失败不影响涨停接口，各自处理异常
2. **日期计算**：昨日日期基于传入的 `trade_date` 计算，不是系统日期
3. **格式标准化**：AKShare/sina 输出转换为与 TuShare 一致的格式
4. **满足需求**：第三步专门满足"昨日涨停今日开盘选股"需求

### 4.3 数据格式标准化转换

**AKShare → TuShare 格式**：

```python
# AKShare 列名 → TuShare 列名
代码 → ts_code (添加 .SZ/.SH 后缀)
名称 → name
最新价 → close
涨跌幅 → pct_chg
振幅 → amplitude
换手率 → turnover
成交量 → volume
成交额 → amount
...
```

## 6. 接口详细设计

### 6.1 ListenerNode 主程序

**文件**：`nodes/listener/node.py`

**入口**：由 main.py 启动，根据 `NODE_TYPE=listener` 进入

**主循环**：

```python
while running:
    1. 如果不是交易时间 → 休眠一轮 (60秒) → 继续
    2. 获取今日日期
    3. 获取涨跌停列表 (经过三级 fallback)
    4. 构建当日快照
       ├─→ 获取涨跌停股票实时行情
       ├─→ 获取历史K线缓存
    5. 遍历所有启用的订阅
       ├─→ 每个策略评估
       ├─→ 收集触发的预警
    6. 对每个触发的预警
       ├─→ 发送 Webhook 通知
       ├─→ 写入飞书多维表格 (如果配置)
       ├─→ 存入 MongoDB
    7. 休眠一轮 (60秒) → 下一轮
```

**配置**：
- 轮询间隔：默认 60 秒
- 交易时间判断：A股交易日 9:15-15:00

  └─→ initialize_all_managers()
