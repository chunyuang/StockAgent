# StockAgent 免费数据源汇总分析报告

## 项目背景

StockAgent 需要稳定的免费 A 股数据接口，用于超短线游资策略的实时选股和历史回测。当前核心需求是涨跌停数据、日线行情、实时行情，需要尽可能整合多个免费数据源，保证数据可用性和稳定性。

---

## 一、核心数据类型需求分析

| 数据类型 | 用途 | 频率 | 必需性 |
|---------|------|------|--------|
| 每日涨跌停列表 | 选股池筛选 | 每日开盘前 | ✅ 必需 |
| 股票日线行情 | 均线计算、形态分析 | 每日更新 | ✅ 必需 |
| 日内实时行情 | 实时监控触发条件 | 盘中每分钟 | ✅ 必需 |
| 股票基础信息 | 代码名称对应、过滤退市 | 静态 | ✅ 必需 |
| 龙虎榜数据 | 龙头战法辅助 | 每日盘后 | ⭐ 可选增强 |
| 资金流向数据 | 强度分析 | 每日 | ⭐ 可选增强 |
| 涨停原因/概念 | 板块分析 | 每日 | ⭐ 可选增强 |
| 历史涨跌停数据 | 回测策略 | 历史 | ⭐ 回测必需 |

## 二、现有数据源评估

### 2.1 TuShare

**官网**: https://tushare.pro/

**接口使用情况**：
- `stk_limit()` - 每日涨跌停数据
- `daily()` - 日线行情
- `daily_basic()` - 每日指标
- `stock_basic()` - 股票基础信息

**优势**：
- 数据质量高，稳定
- API 设计友好，字段规范
- 支持历史数据批量获取

**劣势**：
- 需要积分，新用户积分有限
- 免费用户接口调用限额
- 涨跌停数据获取需要 2000 积分起步

**当前状态**：已集成，作为第一优先级数据源，有权限时使用

**评分**：⭐⭐⭐⭐⭐

---

### 2.2 AKShare

**官网**: https://akshare.akfamily.xyz/

**接口使用情况**：
- `stock_zt_pool_em()` - 今日涨停
- `stock_zt_pool_dtgc_em()` - 今日跌停
- `stock_zt_pool_previous_em()` - 昨日涨停
- `stock_zh_a_daily()` - A股日线

**优势**：
- 完全免费，无需注册
- 无需积分，无调用限制
- 更新活跃，社区支持好
- 数据来源东方财富/同花顺，质量可靠

**劣势**：
- 历史数据获取有限，涨跌停只能获取最近约 30 天
- 某些接口字段不稳定，可能变化
- 数据密度大，解析较慢

**当前状态**：已集成，作为 TuShare 失败后的 fallback

**评分**：⭐⭐⭐⭐

**重要发现**：
- `stock_zt_pool_dtgc_em`（跌停接口）确实只能返回最近 30 日历天数据，更早日期返回空
- 涨停接口`stock_zt_pool_em`同样只返回最近数据
- `stock_zt_pool_previous_em`返回昨日涨停，正好满足 `limit_open` 策略需求

### 2.3 东方财富网络爬虫（新浪/东方财富）

## 五、推荐多数据源整合方案

### 5.1 实时选股服务（当前生产环境）

**推荐四级 fallback 流程**：

```
1. TuShare stk_limit → 成功则使用
   ↓ 失败/空/无权限
2. AKShare 今日涨跌停 → 成功则使用
   ↓ 结果为空/超过30天
3. AKShare 昨日涨停 → 获取昨日涨停，满足今日开盘选股（limit_open需求）
   ↓ 都失败
4. 东方财富爬虫 → 兜底抓取
```

**当前状态**：✅ 已经实现前三级，第四级兜底代码已存在

**满足需求**：✅ 完全满足"昨日涨停今日开盘选股"需求

## 六、分级 fallback 架构设计

StockAgent 当前已经实现了优秀的多数据源 fallback 架构：

```
TushareManager (优先)
  └──→ 失败 → AKShareManager (二级)
          └──→ 结果空 → AKShare 昨日涨停 (三级)
                  └──→ 失败 → SinaSpiderManager (兜底，四级)
```

**优势**：
- 高可用性：一个数据源挂了自动切下一个
- 免费优先：AKShare 完全免费，满足基本需求
- 不丢弃信号：最大限度保证开盘时有股票可筛选

## 八、存在的限制和应对方案

| 限制 | 影响 | 应对方案 |
|------|------|----------|
| AKShare 只能获取近 30 天涨跌停 | 回测更早日期无法获取 | 回测使用 TuShare 积分额度，实时服务不受影响 |
| 免费数据源调用频率限制 | 批量拉取可能被封 | 增加延时，分批次拉取 |
| 网页爬虫可能失效 | 兜底方案失效 | 及时维护选择器，AKShare 已经满足主要需求 |

## 九、结论和建议

### 9.1 当前方案评估

**实时选股服务**：
- 当前三级 fallback 方案已经**完全满足需求**
- 即使 TuShare 无权限，AKShare 也能提供足够数据支持每日开盘选股
- `limit_open` 策略"昨日涨停今日开盘选股"需求已经完美实现

**回测**：
- 近 30 天回测完全没问题
- 更长周期回测需要 TuShare 积分或其他数据源补充
- 推荐集成 Baostock 作为免费日线数据补充

### 9.2 改进建议

1. **新增集成 Baostock**：免费完整日线历史数据，大幅提升回测能力
2. **可选集成龙虎榜**：AKShare 已有接口，龙头战法可以增加龙虎榜筛选条件
3. **每日自动备份涨跌停数据**：保存到 MongoDB，积累历史数据用于回测
4. **保持现有架构**：多数据源 fallback 设计优秀，不需要大改
