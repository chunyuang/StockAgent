## 🔧 修复内容概要

### ❌ 修复前的问题：
- **粗暴性能优化**：强制 `start_dt = end_dt`，只查询1天数据
- **后果**：所有需要历史数据计算的技术因子全部变成 NaN
- **影响范围**：MA5/MA10/MA20/动量/波动率/RSI等13个核心技术因子全部失效
- **实际效果**：选股变成随机选股，所有因子权重形同虚设

### ✅ 修复内容：
1. **回滚代码**：删除强制 `start_dt = end_dt` 的粗暴修复
2. **性能优化1**：创建 MongoDB 复合索引 `(trade_date, ts_code)`
3. **性能优化2**：实现字段投影，只查询需要的因子字段（内存占用减少90%）
4. **验证标准**：因子覆盖率 ≥ 95%，不同因子选股重叠度 < 50%
