## 二、核心基础设施

### 2.1 三级涨跌停数据获取 fallback

为满足"昨日涨停今日开盘选股"需求，设计三级 fallback：

```
1. Tushare stk_limit 接口 → 获取今日涨跌停
   ↓ 失败/无权限/返回空
2. AKShare stock_zt_pool_em + stock_zt_pool_dtgc_em → 获取今日涨跌停
   ↓ 返回空
3. AKShare stock_zt_pool_previous_em → 获取昨日涨停
```

**代码位置**: `core/managers/tushare_manager.py` -> `get_stk_limit()`

**满足需求**: ✅ 当今日涨跌停数据为空时，自动 fallback 获取昨日涨停，满足早盘选股需求。

### 2.2 数据过滤规则

1. **ST 股票过滤**: 自动过滤名称包含 `ST`, `*ST`, `S*ST`, `SST` 的股票
2. **数据库有效性过滤**: 只保留 MongoDB `stock_basic` 中存在且状态为上市的股票
3. **涨跌停价格**: 从接口获取准确的涨跌停价格，用于开板判断

### 3.1 涨跌停开板策略 (limit_open)

**代码文件**: `nodes/listener/strategies/limit_open.py`  
**策略类型**: 超短打板 / 翘板

#### 策略逻辑

> 捕捉涨停/跌停开板后的资金接力机会。

| 子策略 | 适用场景 |
|--------|----------|
| **涨停开板** | 涨停板炸开，博弈回封或次日高开 |
| **跌停翘板** | 跌停板撬开，博弈反弹 |

#### 入场因子

| 因子 | 判断规则 |
|------|----------|
| **封板状态** | 上一轮询价格 ≈ 涨跌停价 |
| **开板状态** | 当前轮询价格 偏离 涨跌停价 |
| **参数控制** | `limit_type`: "up" / "down" / "both" |

#### 检测流程

```python
# 涨停打开判断
was_at_limit = prev_price >= up_limit
is_opened = current_price < up_limit
return was_at_limit and is_opened

# 跌停打开判断
was_at_limit = prev_price <= down_limit
is_opened = current_price > down_limit
return was_at_limit and is_opened
```

#### 触发输出字段

| 字段 | 说明 |
|------|------|
| `limit_type` | "up" 涨停开板 / "down" 跌停开板 |
| `up_limit` / `down_limit` | 涨跌停价格 |
| `prev_price` | 上一周期价格 |
| `pct_chg` | 当前涨跌幅 |

#### 适用场景

- 涨停开板：龙头股分歧转一致
- 跌停翘板：超跌股错杀反弹

### 3.2 5日线低吸策略 (ma5_buy)

**代码文件**: `nodes/listener/strategies/ma5_buy.py`  
**策略类型**: 龙头低吸 / 均线回踩

#### 策略逻辑

> 龙头股经过一波上涨后，回落到5日线企稳时低吸，享受五日均线支撑后的二次上涨。

#### 核心因子

| 因子 | 计算方式 | 说明 |
|------|----------|------|
| **MA5 计算** | `(前4天收盘价 + 今日开盘价 * 1.05) / 5` | 盘中固定，不随实时价格变动 |
| **距离** | `(当前价 / MA5) - 1` | 计算当前价偏离5日线的百分比 |
| **触及范围** | `abs(距离) <= touch_range` | 默认 `touch_range = 0.02` (±2%) |
| **企稳** | 连续 N 个周期在 5日线上方 | 默认 `stable_periods = 2` |
| **昨日位置** | `prev_close > MA5` | 要求昨日在5日线上方 |

#### 状态机逻辑

```
状态0: NORMAL
  ↓ 满足条件: 昨日在MA5上方 + 今日回落到MA5附近
状态1: TOUCHED (已触及)
  ↓ 价格在MA5上方
    stable_count += 1
    ↓ stable_count >= stable_periods
状态2: STABILIZED (企稳确认) → 触发信号
```

#### 参数配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `touch_range` | 0.02 | 触及范围 ±2% |
| `stable_periods` | 2 | 需要连续几个周期站稳 |
| `once_per_day` | True | 每只股票每天只触发一次 |

#### 触发输出字段

| 字段 | 说明 |
|------|------|
| `ma5` | 5日均线价格 |
| `prev_close` | 昨日收盘价 |
| `distance_to_ma5` | 当前价偏离百分比 |
| `stable_count` | 企稳连续周期数 |
| `touch_time` | 首次触及时间 |

#### 适用场景

- 龙头股回踩低吸
- 趋势股均线支撑
- **对应策略分类**: 龙头低吸

### 3.3 涨跌幅阈值策略 (price_change)

**代码文件**: `nodes/listener/strategies/price_change.py`  
**策略类型**: 半路追涨 / 急速跳水

#### 策略逻辑

> 当股票涨跌幅超过设定阈值时触发预警，用于捕捉半路冲高或急速跳水。

#### 核心因子

| 因子 | 判断规则 |
|------|----------|
| **涨幅阈值** | `pct_chg >= threshold` | 上涨突破阈值 |
| **跌幅阈值** | `pct_chg <= -threshold` | 下跌跌破阈值 |
| **方向控制** | `direction: up/down/both` | 监控方向 |
| **每日一次** | `once_per_day: True` | 每日只触发一次 |

#### 参数配置

| 参数 | 说明 | 示例 |
|------|------|------|
| `threshold` | 涨跌幅阈值 (百分比) | `5.0` 表示 5% |
| `direction` | 监控方向 | `up` = 只监控涨 |
| `once_per_day` | 每日只触发一次 | `True` |

#### 触发输出字段

| 字段 | 说明 |
|------|------|
| `pct_chg` | 当前涨跌幅 |
| `threshold` | 阈值 |
| `direction` | 方向 |
| `vol` / `amount` | 成交量 / 成交额 |

#### 适用场景

- **半路追涨**: 盘中涨幅突破 5%/7% 追入
- **跳水监控**: 跌幅突破 5%/7% 监控恐慌机会

### 3.4 龙头战法 (leading_dragon)

**代码文件**: `nodes/listener/strategies/leading_dragon.py`  
**策略类型**: 最高连板龙头捕捉

#### 策略规则（来自飞书多维表格）

> 捕捉市场最高连板的情绪龙头，享受全市场资金聚焦的情绪溢价，属于强者恒强的最高阶玩法。

**入场条件**:
1. ✅ 连板高度为市场当前最高
2. ✅ 换手充分（非一字板上来，最好有爆量换手板）
3. ✅ 所属板块为当前市场主线，有板块效应支撑
4. ✅ 情绪周期处于上升期或发酵期

**出场条件**:
1. ✅ 断板无法回封时止盈
2. ✅ 收盘价跌破5日均线止损
3. ✅ 出现明显的情绪退潮信号时无条件离场

**仓位管理**:
- 单票仓位不超过总资金的 **20%**
- 龙头确定后可分 **2-3次加仓**
- 总仓位不超过 **30%**

#### 核心因子实现

| 因子 | 判断规则 | 默认值 |
|------|----------|--------|
| **连板高度** | 连续涨停天数 >= `min_height` | `min_height = 3` |
| **换手率** | 当日换手率 >= `min_turnover` | `min_turnover = 15%` |
| **当前涨停** | `current_price >= up_limit` | - |
| **昨日涨停** | 昨日已涨停（计算连板） | - |

#### 连板高度计算

```python
height = 1
if 昨日已涨停:
    height += 1
if height >= min_height → 通过
```

#### 参数配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `min_height` | 3 | 最低连板高度 |
| `min_turnover` | 15 | 最低换手率 (%) |

#### 触发输出字段

| 字段 | 说明 |
|------|------|
| `height` | 连板高度 |
| `turnover` | 换手率 |
| `up_limit` | 涨停价 |
| `pct_chg` | 当前涨跌幅 |

#### 匹配策略分类

✅ **完全匹配** 多维文档中 "龙头战法" 分类

### 3.5 首板打板 (first_board)

**代码文件**: `nodes/listener/strategies/first_board.py`  
**策略类型**: 首次涨停博弈

#### 策略规则（来自飞书多维表格）

> 在股票第一次涨停瞬间买入，博弈次日的溢价或连板，属于低风险高胜率的打板入门玩法。

**入场条件**:
1. ✅ 股价首次触及涨停价
2. ✅ 涨停封单量足够（通常大于流通市值的 1%）
3. ✅ 所属板块有异动，最好有板块内其他个股助攻
4. ✅ 成交量相较于前几日明显放大，换手充分

**出场条件**:
1. ✅ 次日高开 3% 以上可分批止盈
2. ✅ 次日开盘不及预期（低开或平开），冲高无力时止盈
3. ✅ 买入当日炸板无法回封，次日无条件止损

**仓位管理**:
- 单票仓位 **5%-10%**
- 同时打 **3-5只** 首板分散风险
- 总仓位不超过 **30%**

#### 核心因子实现

| 因子 | 判断规则 | 默认值 |
|------|----------|--------|
| **触及涨停** | `current_price >= up_limit * 0.99` | 允许 1% 误差 |
| **换手率** | `turnover >= min_turnover` | `min_turnover = 8%` |
| **成交量放量** | `volume / volume_5d_avg >= min_volume_ratio` | `min_volume_ratio = 1.5x` |

#### 参数配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `min_turnover` | 8 | 最低换手率 (%) |
| `min_volume_ratio` | 1.5 | 最低成交量相对于 5 日均量的倍数 |

#### 触发输出字段

| 字段 | 说明 |
|------|------|
| `up_limit` | 涨停价 |
| `turnover` | 换手率 |
| `volume_ratio` | 成交量倍数 |
| `pct_chg` | 当前涨跌幅 |

#### 匹配策略分类

✅ **完全匹配** 多维文档中 "首板打板" 分类

## 五、数据完整流程

```
交易日 9:00 
  ↓
获取涨跌停价格 (三级 fallback)
  ↓
每分钟轮询:
  ↓
  1. 获取需要监听的股票列表
     (全市场监听 → 使用涨跌停列表；指定个股 → 使用订阅列表)
  ↓
  2. 分批获取实时行情 (每批 50 只)
  ↓
  3. 构建市场快照
  ↓
  4. 存储实时行情到 Redis (供 Web 端访问)
  ↓
  5. 遍历所有策略订阅 → 执行 evaluate()
  ↓
  6. 触发信号:
     → Webhook 推送到飞书群聊
     → 写入飞书多维表格
  ↓
等待下一分钟
```
