## 1. 迁移前准备

迁移前，请确认已备份以下内容：

| 内容 | 位置 | 是否必须 |
|------|------|---------|

| `.env` 配置文件 | `StockAgent/AgentServer/.env` | ✅ 必须 (包含token) |

## 2. 迁移步骤

### 步骤 1：复制源码

```bash
# 在目标服务器
cd /path/to/workspace
tar xzf StockAgent-v0.2.tar.gz
cd StockAgent
```

### 步骤 2：安装依赖

```bash
pip install -r requirements.txt
# 如果系统提示外部环境错误
pip install -r requirements.txt --break-system-packages
```

**确认关键包安装成功**：
```bash
pip list | grep -E "akshare|baostock|tushare|redis|pymongo"
```

应该能看到：
- `akshare` 版本 ≥ 1.10
- `baostock` 版本 ≥ 0.8
- `tushare` 版本 ≥ 1.2
- `redis` 版本 ≥ 4.0
- `pymongo` 版本 ≥ 4.0

### 步骤 3：还原配置

```bash
# 将备份的 .env 复制到
cp your-backup/.env /path/to/StockAgent/AgentServer/.env
```

**检查 `.env` 配置**：
- `REDIS_HOST`、`REDIS_PORT`、`REDIS_PASSWORD` 符合新环境
- `MONGO_HOST`、`MONGO_PORT` 符合新环境
- `TUSHARE_TOKEN` 已复制
- `FEISHU_APP_ID`、`FEISHU_APP_SECRET` 已复制（如果使用多维表格）
- `NOTIFY_WECOM_WEBHOOK` 已复制

### 步骤 4：还原 MongoDB 数据

```bash
# 如果你备份了 mongodb 数据
mongorestore -d stock_agent /path/to/backup/stock_agent/
```

如果是全新部署，不需要提前还原，程序启动时会自动创建索引。

### 步骤 5：启动服务

```bash
cd /path/to/StockAgent/AgentServer
mkdir -p logs
export NODE_TYPE=listener
python3 main.py > logs/listener.log 2>&1 & disown
```

### 步骤 6：检查运行状态

等待 10 秒，查看日志：

```bash
tail -50 logs/listener.log
```

应该看到：
```
Redis connected ✓
MongoDB connected ✓
Tushare initialized ✓
Baostock 登录成功 ✓
Registered strategies: ['limit_open', 'price_change', 'ma5_buy', 'leading_dragon', 'first_board']
Loaded 146 active subscriptions
gRPC server started on 0.0.0.0:50053
```

## 5. 故障排查

### 问题 1：启动失败，找不到模块

```
ModuleNotFoundError: No module named 'baostock'
```

**解决**：
```bash
pip install baostock --break-system-packages
```

### 问题 2：Redis 连接失败

```
Connection refused to Redis
```

**解决**：
```bash
# 启动 Redis
redis-server --daemonize yes
# 检查
redis-cli ping
```

### 问题 3：MongoDB 连接失败

```
Connection refused to MongoDB
```

**解决**：
```bash
# systemd
systemctl start mongod
# 或
mongod --fork --logpath /var/log/mongodb.log --dbpath /var/lib/mongodb
```

### 问题 4：AKShare 返回空涨跌停

**原因**：获取超过 30 天前的数据，AKShare 接口限制

**解决**：
- 这是正常现象，不是 bug
- 实时选股只需要今日/昨日，不受影响
- 回测需要历史数据，请使用 TuShare 或 Baostock 获取日线

### 问题 5：listener 启动了但没有信号

**排查步骤**：
1. 检查是否在交易时间 → 非交易时间不获取数据，正常
2. 检查日志看涨跌停获取是否成功 → `grep "get_stk_limit" logs/listener.log`
3. 检查是否有订阅策略 → `Loaded XXX active subscriptions` 应该大于 0
4. 检查 MongoDB 中 `strategy_subscriptions` 集合 → 确认有启用的订阅

### 问题 6：飞书多维表格写入失败

**排查**：
1. 检查 `FEISHU_APP_ID` 和 `FEISHU_APP_SECRET` 是否正确
2. 检查飞书应用是否有 `bitable:app:read` 和 `bitable:app:write` 权限
3. 检查多维表格是否对应用开放 → 添加应用为协作者
4. 检查 `FEISHU_BITABLE_APP_TOKEN` 和 `FEISHU_BITABLE_TABLE_ID` 是否正确
