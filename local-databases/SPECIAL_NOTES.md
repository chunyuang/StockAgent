# 数据源特殊注意事项

记录每个数据源存在的特殊数据问题，避免重复踩坑。

---

## Tushare

### 优点
- 数据质量最高，复权准确，分红拆股处理完整
- 基本面数据非常齐全

### 特殊注意
- 需要积分，积分不够有些数据拿不到
- Pro 需要付费
- 请求频率有限制，不能太快

### 适合
- 全历史回测，积分足够的时候优先用

---

## AKShare

### 优点
- 完全免费，不需要 Token
- 无请求频率限制
- 数据更新及时

### 特殊注意
- 复权处理有时候不如 Tushare 准确
- 基本面数据不如 Tushare 全

### 适合
- 免费替代，不需要 Token，回归测试首选
- 近期数据获取

---

## 比盈 (Biying)

### 优点
- 数据质量高，接口稳定
- 免费额度 200 次/天，够用实盘

### 特殊注意
- 需要注册获取 Token
- 每日额度有限，超过了要等第二天
- 不适合全量下载几万条数据

### 适合
- 实盘实时行情获取
- 少量数据补充

---

## Baostock

### 优点
- 完全免费，不需要 Token
- 复权价格准确

### 特殊注意
- 更新比较慢，最新数据可能延迟
- 只支持日线，不支持分钟线

### 适合
- 免费历史数据获取备用

---

## MongoDB 本地 fallback

### 用途
涨跌停价格如果在线数据源拿不到，本地计算

### 公式
- 涨停价 = 昨收 × 1.1
- 跌停价 = 昨收 × 0.9
- ST 股 × 1.05 / 0.95

## 推荐最佳实践

### 全量下载大时间段

```
分片并行获取：
- Tushare 负责 2010-2018
- AKShare 负责 2019-现在
同时并行跑，比一个一个快很多
```

### 实盘每日更新

```
优先级：
比盈 → AKShare → Baostock → MongoDB
每天只需要获取最新一个交易日，额度够用
```

### 增量更新

```
直接用 AKShare 获取最新交易日数据，免费无限制
增量更新后自动更新 metadata 和 dashboard
```
