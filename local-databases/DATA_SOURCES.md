## 并行获取优化

### 并行获取 vs 线性 fallback

**原来的方式（线性 fallback）：**
```
一个股票 → 尝试Tushare → 失败等重试 → 尝试AKShare → 失败 → 尝试比盈
→ 下一个股票
```
❌ 慢，一个股票失败要等很久

**优化后的方式（并行获取）：**
```
同时发起多个数据源请求 → 哪个先返回成功就用哪个 → 取消其他还在等待的请求
多个股票并行处理 → 整体速度快很多
```
✅ 快，充分利用带宽

### 数据源分片（每个数据源负责不同时间段）

可以配置每个数据源负责自己擅长的时间段：
```
Tushare → 2010-01-01 ~ 2018-12-31
AKShare → 2019-01-01 ~ 现在
```
这样不同数据源并行获取不同时间段，最后合并，速度更快。

## 使用规则与最佳实践

### 回测场景

**最佳实践**:
1. ✅ **优先用本地缓存**：已经下载好的数据存在 `local-databases`，直接用本地，不用重复请求
2. ✅ **数据一致性**：回测前自动校验，保证数据条数和记录一致，避免因为数据不一致导致回测错误
3. ✅ ** fallback 链条**：配置多个 fallback，第一个失败自动切下一个

**当前 fallback 链条** (已经实现):
```
Tushare → AKShare → Baostock → MongoDB 本地计算（涨跌停）
```

### 实盘/实时场景

**最佳实践**:
1. ✅ **优先用比盈**：质量高，免费额度够用（200次/天，实盘每天只需要请求一次够用）
2. ✅ **备用 AKShare**：比盈不行的时候切 AKShare
3. ✅ **不要频繁请求**：cached 本地已经有的，不用重复请求
4. ✅ **控制频率**：即使免费数据源也不要过于频繁请求，避免被封禁

## 各个数据源注意事项

### Tushare

- 需要注册获取 Token，积分限制
- 积分可以通过签到获取，够用基础回测
- Pro 版本需要付费，数据更全
- 如果积分够，优先用 Tushare，数据质量很好
- **替代**: AKShare 完全免费，可以替代不需要积分

### AKShare

- **完全免费**，不需要 Token
- 数据覆盖全面，更新及时
- 无请求频率限制，可以随便用
- 数据质量略逊色于 Tushare/比盈，但够用回归测试
- **推荐**: 开源免费首选，回归测试完全够用

### 比盈 (Biying)

- 需要注册获取 Token
- 免费额度 **200 次/天**，对于实盘完全够用
- 数据质量高，接口稳定
- 推荐用于**实时行情获取**，免费额度够用
- 如果超过次数，自动 fallback 到 AKShare

### Baostock

- 完全免费，不需要 Token
- 数据更新较慢，只能获取历史数据
- 推荐用作**免费历史数据**备用

## 如何添加新数据源

1. 在 `DATA_SOURCES.md` 这里添加记录
2. 在代码中实现 `xxx_manager.py`，实现 `get_daily()` 接口
3. 在 fallback 链条中添加，修改 `__init__.py`
4. 保证遵循 fallback 规则，自动切换

## 避坑指南

1. **不要重复请求相同数据**：已经下载到本地的，直接用本地，存在 `local-databases` 里面，校验过一致性，放心用
2. **控制请求频率**：即使免费接口也不要疯狂请求，容易被封禁
3. **保留 fallback**：永远配置多个数据源，一个不行自动切下一个
4. **本地校验**：每次回测前自动校验数据一致性，避免"因子改了没重新计算"问题
5. **分开存储**：原始数据/因子/回测结果三级分开，互不干扰，改哪级只需要重新计算哪级
